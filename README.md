# 📚 История на пальцах — навигация + админ-панель

Электронный каталог школьных материалов по истории (5–11 классы) с публичной
страницей-навигацией и **админ-панелью**, через которую владелец канала MAX
сам добавляет темы и ссылки — без программирования.

- **Публичная навигация:** `https://ruslan709.github.io/history/`
- **Админ-панель:** `https://ruslan709.github.io/history/admin.html`

Стек: **React + TypeScript + Vite**. Данные — один JSON-файл в репозитории
(`public/data/content.json`). Сервер не нужен: админка сохраняет изменения
прямо в GitHub через его API, а GitHub Pages автоматически пересобирает сайт.

---

## 🗂️ Как устроены данные

```
Класс (5–11)
 └─ Раздел / Глава (Глава I. Первобытное общество)
     └─ Тема (§1. Древнейшие люди)  →  ссылка на пост в MAX
```

База уже заполнена **реальными оглавлениями учебников** (5–11 классы,
всеобщая история + история России, 46 глав, ~290 тем). Ссылки на MAX пустые —
владелец проставляет их в админке.

## 🚀 Развёртывание (делается один раз)

1. Скопируйте содержимое этой папки в репозиторий `ruslan709/history`
   (можно заменить старые файлы — навигация теперь генерируется отсюда).
2. Отправьте в GitHub:
   ```bash
   git add .
   git commit -m "Навигация + админ-панель"
   git push
   ```
3. В репозитории откройте **Settings → Pages** и в разделе **Build and deployment**
   выберите **Source: GitHub Actions**.
4. Через 1–2 минуты сайт соберётся и откроется по адресу
   `https://ruslan709.github.io/history/`.

> Если репозиторий называется не `history`, поменяйте `base` в `vite.config.ts`
> и константы в `src/config.ts`.

## 🔑 Настройка админки (делает разработчик, один раз)

Заказчик заходит только по **логину и паролю** и токен не видит. Токен
настраивается один раз на устройстве заказчика.

**Данные входа по умолчанию:** логин `admin`, пароль `istoriya2026`.

1. Создайте токен GitHub: `github.com` → **Settings → Developer settings →
   Personal access tokens → Fine-grained tokens → Generate new token**.
   - **Repository access** → Only select repositories → `history`.
   - **Permissions → Repository → Contents** → `Read and write`.
   - Скопируйте токен (`github_pat_…`).
2. Откройте `…/history/admin.html`, войдите (`admin` / `istoriya2026`),
   вкладка **⚙️ Настройки**:
   - В разделе **«Вход администратора»** задайте логин и пароль для заказчика.
   - В разделе **«Настройка сохранения»** вставьте токен → «Сохранить и проверить».
3. **Настройка устройства заказчика:**
   - Если это его телефон/компьютер — просто вставьте токен здесь же (готово).
   - Если удалённо — нажмите **«🔗 Ссылка-настройка для заказчика»**, отправьте
     ссылку заказчику. Он один раз её открывает — токен сохраняется в его
     браузере. Дальше он заходит только по логину и паролю.

Токен хранится только в браузере (в `localStorage`), в коде и в репозитории его нет.

## 👉 Как заказчику добавить материал (каждый день)

Никаких токенов и настроек — просто:

1. Открыть `…/history/admin.html`, ввести **логин и пароль**.
2. Вкладка **«Обзор» → «Добавить тему»**: выбрать класс и раздел, вписать
   название, выбрать иконку, вставить ссылку на пост MAX → **«Сохранить тему»**.
3. Нажать **«💾 Сохранить на сайт»** вверху справа.
4. Через 1–2 минуты навигация обновится автоматически.

Также во вкладке **«Каталог»** можно редактировать, удалять и менять порядок
классов, разделов и тем; добавлять новые классы и главы.

## 🔒 О безопасности

Пароль администратора проверяется в браузере и защищает от случайных заходов.
Реальная защита записи — токен GitHub: без него сохранить изменения нельзя.
Для одного владельца-редактора этого достаточно. Если позже понадобится
несколько редакторов или серверная авторизация — данные лежат в обычном JSON,
поэтому проект легко перенести на Node/SQLite или облачную функцию.
